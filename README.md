# QA Visual Checker

Набор инструментов для автоматической проверки сайтов. Помогает тестировщику быстро находить проблемы без ручной проверки каждой страницы.

---

## Что умеет этот инструмент

| Инструмент | Что проверяет | Команда запуска |
|---|---|---|
| **Сравнение с макетом** | Совпадает ли страница с дизайном в Figma | `python runner.py` |
| **Проверка ссылок** | Нет ли битых ссылок и недоступных картинок | `python run_checks.py --check links` |
| **Проверка консоли** | Нет ли JS-ошибок на странице | `python run_checks.py --check console` |
| **Проверка адаптивности** | Как страница выглядит на разных устройствах | `python run_responsive.py` |

---

## Установка (один раз)

### Шаг 1 — Установи Python

Открой сайт https://python.org/downloads и скачай Python 3.11 или новее.
При установке обязательно поставь галочку **"Add Python to PATH"**.

Проверь что всё установилось — открой Терминал и напиши:
```
python --version
```
Должно написать что-то вроде `Python 3.11.4`.

### Шаг 2 — Скачай проект

Положи папку с проектом куда удобно, например на Рабочий стол.

### Шаг 3 — Открой папку проекта в Терминале

На Mac: открой Терминал, напиши `cd ` (со пробелом), затем перетащи папку проекта прямо в окно терминала и нажми Enter.

На Windows: открой папку проекта в проводнике, зажми Shift и кликни правой кнопкой мыши → "Открыть окно PowerShell здесь".

### Шаг 4 — Создай виртуальное окружение

Это нужно чтобы библиотеки проекта не конфликтовали с другими программами.

```
python -m venv venv
```

Затем активируй его:

На Mac/Linux:
```
source venv/bin/activate
```

На Windows:
```
venv\Scripts\activate
```

После активации в начале строки терминала появится `(venv)` — это значит всё правильно.

### Шаг 5 — Установи библиотеки

```
pip install -r requirements.txt
playwright install chromium
```

Это может занять 2-3 минуты — просто подожди.

### Шаг 6 — Создай файл с ключами

Создай файл `.env` в папке проекта. Открой любой текстовый редактор (Блокнот, TextEdit) и напиши:

```
FIGMA_TOKEN=вставь_сюда_свой_токен_figma
```

Сохрани файл с именем `.env` (именно с точкой в начале) в папку проекта.

Где взять Figma токен: открой Figma → кликни на своё имя в левом верхнем углу → Settings → Security → Personal access tokens → Generate new token. Скопируй токен и вставь в `.env`.

## Запуск проверок

> Перед каждым запуском убедись что в терминале активировано окружение — в начале строки должно быть `(venv)`. Если нет — выполни команду активации из Шага 4.

---

### Проверка 1 — Сравнение страницы с макетом Figma

Сравнивает скриншот реальной страницы с дизайном из Figma и показывает где есть расхождения.

**Настройка**

Открой файл `config.yaml` в текстовом редакторе. Он выглядит так:

```yaml
pages:
  - name: login-page
    url: https://твой-сайт.com/страница
    figma_file_key: aBcDeFgHiJkL
    figma_node_id: "12:345"
```

Замени значения на свои:

- `name` — любое название для отчёта, например `главная` или `страница-входа`
- `url` — адрес страницы которую хочешь проверить
- `figma_file_key` — часть URL твоего Figma файла: `figma.com/design/ВОТЭТА_ЧАСТЬ/название`
- `figma_node_id` — ID фрейма в Figma: кликни правой кнопкой на фрейм → Copy link → в ссылке найди `node-id=12-345` → запиши как `"12:345"` (дефис меняй на двоеточие)

**Запуск**

```
python runner.py --config config.yaml
```

Первый раз макет скачивается из Figma. Следующие запуски используют сохранённую копию — быстрее и без лимитов API.

Если дизайнер обновил макет — добавь флаг чтобы перескачать:
```
python runner.py --config config.yaml --refresh-figma
```

**Результат**

Открой файл `output/report.html` в браузере. Там будет три картинки рядом: макет, страница, и diff где красным отмечены расхождения.

### Проверка 2 — Битые ссылки и недоступные картинки

Открывает страницу, находит все ссылки и картинки, и проверяет каждую — отвечает ли она нормально.

**Запуск**

```
python run_checks.py --url https://твой-сайт.com --check links
```

Замени `https://твой-сайт.com` на адрес страницы которую хочешь проверить.

**Результат**

Открой `output/report_links.html`. Увидишь таблицу со всеми ссылками:
- зелёный статус — ссылка работает
- красный статус — ссылка сломана (404 = страница не найдена, 500 = ошибка сервера, ERR = сервер не отвечает)

---

### Проверка 3 — JS-ошибки в консоли

Открывает страницу и собирает все ошибки и предупреждения из консоли браузера — то же самое что ты видишь в DevTools → Console, только автоматически.

**Запуск**

```
python run_checks.py --url https://твой-сайт.com --check console
```

**Результат**

Открой `output/report_console.html`. Там будет список:
- **error** — настоящие JS-ошибки, нужно исправлять
- **pageerror** — необработанные исключения, критично
- **request_failed** — запросы которые упали (кроме рекламных — они блокируются намеренно)
- **warning** — предупреждения, стоит изучить

---

### Проверка 4 — Адаптивность на разных устройствах

Открывает страницу на 5 разных размерах экрана и автоматически ищет проблемы верстки на каждом.

**Запуск**

```
python run_responsive.py --url https://твой-сайт.com
```

**Результат**

Открой `output/report_responsive.html`. Для каждого устройства увидишь:
- скриншот страницы
- список найденных проблем

Проверяемые устройства:
- Mobile S — 320px (старые маленькие телефоны)
- Mobile L — 375px (iPhone и большинство Android)
- Tablet — 768px (iPad и планшеты)
- Laptop — 1280px (ноутбуки)
- Desktop — 1440px (мониторы)

Что ищет автоматически:
- горизонтальный скролл (страница шире экрана)
- картинки которые вылезают за края
- текст который обрезается
- шрифты меньше 12px (нечитаемо)
- кнопки слишком маленькие для нажатия пальцем
- отсутствие мобильного меню (бургера) на телефоне

---

### Запустить сразу все проверки (кроме Figma)

```
python run_checks.py --url https://твой-сайт.com
```

Запустит проверку ссылок и консоли, создаст два отдельных отчёта.

```
├── runner.py             — сравнение с макетом Figma

├── console_checker.py    — проверка консоли (не трогать)

│
├── config.yaml           — настройки для сравнения с Figma (редактировать)
├── .env                  — ключи доступа (редактировать, не публиковать!)

└── output/               — сюда сохраняются все отчёты

    ├── report_console.html   — отчёт консоли
    └── report_responsive.html — отчёт адаптивности
```

## Частые проблемы

**`python: command not found`**
Python не установлен или не добавлен в PATH. Переустанови Python с галочкой "Add Python to PATH".

**`(venv)` не появляется**
Виртуальное окружение не активировано. Выполни команду активации из Шага 4.

**`ModuleNotFoundError`**
Библиотеки не установлены. Выполни `pip install -r requirements.txt`.

**`FIGMA_TOKEN не установлен`**
Нет файла `.env` или он лежит не в той папке. Убедись что `.env` лежит в корне проекта рядом с `runner.py`.

**`Rate limit exceeded` от Figma**
Превышен лимит запросов к Figma API (10 в минуту). Подожди 1 минуту и запусти снова. При повторных запусках используй кэш — не добавляй `--refresh-figma` без необходимости.

**Браузер открывается и сразу закрывается**
Это нормально — Playwright открывает браузер, делает что нужно и закрывает. Не пугайся.
